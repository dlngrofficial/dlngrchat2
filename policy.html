gavel Rules & Behavioral Standards

Accessing the DLNGR Chat interface requires adherence to callsign regulations, moderation standards, and node stability policies. Please read these terms carefully.

RULE_01 //

Handle & Callsign Decency

User callsigns (handles) must be between 2 and 24 characters and may contain letters, numbers, dots, spaces, or hyphens. Profanity filtration automatically rejects inappropriate language. Attempting to bypass the filtration using character lookalikes or homoglyphs is strictly prohibited.

RULE_02 //

Authenticity & Anti-Masquerade

Do not attempt to masquerade as another user. Your Identity Code is cryptographic proof of your session's unique identity. Making this code public appends a green verification tick to your callsign. Always verify users' identities through their public tokens.

RULE_03 //

Channel Moderation & Reports

Every message on the network can be reported. Flagging a message via the "Report" option adds the target to the administrative dashboard queue. Falsely reporting messages in bulk is considered a denial-of-service attack on nodes and will lead to an immediate ban.

RULE_04 //

Abuse & Security Enforcement

Administrators (orange badges) and Creators (red badges) have node administrative powers. They can mute users temporarily, block access codes, or issue permanent IP bans to preserve room comfort and safety.

extension Detailed Feature Guide

Explore the full suite of cybernetic features built into DLNGR Chat designed to provide security, speed, and real-time interaction.

keyIdentity Codes & Verification

Upon connection, a random identity hash is assigned to your session. If you choose "Stay Logged In", this session code is securely cached in your browser. Clicking "Show My Code" displays your secret key. You can toggle public verification to append a clean green tag to your callsign, confirming you are the original owner of that handle.

micSpeech Transmission (Speech-to-Text)

Tap the microphone button in the bottom bar to launch the Speech Transmission dialog. Once launched, tap the glowing microphone button inside the popup to record your speech. Your speech is transcribed in real-time into an editable preview textarea. When satisfied, click "Transmit" to post the compiled draft into the main channel instantly.

searchAdvanced Message Search

Click the search icon in the bottom toolbar (or press `/`) to open the search bar at the top of the chat area. Enter keyword queries. The search engine dynamically highlights matching text cards in translucent red and shows the match count (e.g., 2/15). Use the up/down arrows overlay inside the input to cycle and scroll automatically to each matching message. Click the cross "X" button to hide the bar and clear highlighted states.

paletteTheme Customizer

Access the settings panel to configure custom colors. You can select pre-compiled aesthetic dots (Deep Blue, Forest Dark, Terracotta Dark) or pick a custom hexadecimal color for the sidebars, chat screen, and text elements. The Customizer saves your preference in your local cache, restoring your selected dashboard layout at login.

account_circleProfile Avatars

Choose from a curated collection of 17 custom animal and symbol workspace avatar icons, or load the classic blue grid fallback graphic. Clicking any avatar grid cell automatically sets the avatar, broadcasts the update to Firestore presence channels, and slides you back to the main settings page.

add_reactionLive Message Reactions

On desktop, hover over a message card to reveal the floating emoji reaction menu. On mobile, tap on a message to bring up actions. Tap 😂, 👍, or 👎 to react instantly. Click the plus icon on the toolbar to open the search-integrated emoji picker modal. Search emojis dynamically or select from standard category grids to add live counters.

security Node Security & Enforcement

The network relies on decentralized moderation capabilities to flag and purge offensive content without bypassing server rules.

reportEnforcement Reporting System

Clicking "Report" next to any message triggers a confirmation pop-up. Proceeding creates an administrative warning document associated with the message directly within the database stream. This securely bypasses Firestore rules by using the permissive channel stream, preventing server blocks.

admin_panel_settingsModerator Dashboard

Creators and Room Admins see an alert badge next to the theme switch when reported messages are active. Clicking the badge opens the "Reported Messages Dashboard" modal. From this dashboard, admins can review reported logs, moderate users (mute or ban), and dismiss logs from the queue.

warning
